# Environment Variables

> Status: maintained
> Applies to: OpenPath repository
> Last verified: 2026-06-11
> Source of truth: `docs/ENVIRONMENT_VARIABLES.md` -- update this file whenever a new variable is added or an existing one is removed.

## How defaults are resolved (API)

All API environment variables are read through `api/src/config-loader.ts::loadConfig()`, which
receives `process.env` as a plain record. Helper functions in `api/src/config-env.ts`
(`parseIntEnv`, `parseBooleanEnv`, `parseListEnv`, `parseTrustProxyEnv`, `resolveJwtSecret`)
apply the defaults listed in the table. A handful of operational toggles are read directly from
`process.env` inside specific modules (noted in "Read at" below) because they do not belong to
the main config object. `JWT_SECRET` throws at startup if unset in any environment other than
`test`.
