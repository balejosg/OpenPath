# OpenPath Firefox Source Review Notes

OpenPath is a managed education allowlist system. This Firefox extension is the
browser companion for OpenPath-managed devices and is distributed as an
AMO-signed, self-hosted extension.

## How To Rebuild

From the repository root on a local Node/npm environment:

```bash
npm ci
npm run build --workspace=@openpath/firefox-extension
npm run build:firefox-source --workspace=@openpath/firefox-extension
npm run verify:amo-submission --workspace=@openpath/firefox-extension
```

The source submission archive is written to
`firefox-extension/build/firefox-source-submission/openpath-firefox-source.zip`.
The signed runtime bundle contains compiled files under `dist/`; the TypeScript
source for those files is under `firefox-extension/src/`.

`npm ci` uses the checked-in `package-lock.json`. The build uses the local
TypeScript compiler and repository scripts; it does not require web-based build
tools. `dist/` is generated by `npm run build --workspace=@openpath/firefox-extension`
from the TypeScript sources. The source package intentionally excludes
generated `dist/`, `build/`, `node_modules/`, coverage output, and TypeScript
build-info files because they are reproducible from the included sources and
lockfile.

The project does not intentionally obfuscate or minify extension source. The
compiled JavaScript is generated by TypeScript from `firefox-extension/src/`.

## Runtime Behavior

- Unblock requests ask for Firefox's optional `browsingActivity` data-collection
  permission before sending the blocked domain, user-provided reason, origin
  page context when available, and OpenPath request metadata only to the
  configured OpenPath service.
- Google game blocking is local browser policy. The background runtime blocks
  Google Snake, `doodles.google` game frames, and known interactive Google
  logo-game assets through `webRequest` before those game surfaces load.
- The Google Search/Doodles visual guard is a content script scoped to Google
  Search and Doodles hosts. It locally neutralizes detected playable game
  widgets and sends only a best-effort local diagnostic message to the
  extension background runtime.
- Firefox Core registers an isolated-world page activity relay and a MAIN-world
  page-resource observer as content scripts. They report page activity/resource
  candidates to the extension background runtime for local OpenPath policy
  visibility, but they do not upload browsing data automatically and do not
  include automatic allowlist changes.
- Firefox Core does not register automatic AJAX/page-resource allowlisting or
  live/automatic AMO upload. It also does not include Android support.
- `nativeMessaging` is local-only. It contacts the OpenPath native host on the
  same managed computer to inspect or refresh local OpenPath allowlist state.
  The included host supports local actions such as reading blocked paths,
  reading blocked subdomains, checking allowlist state, and requesting a local
  policy refresh from the installed OpenPath client. It does not talk to
  arbitrary native applications.

## Data Boundaries

The extension does not contain third-party analytics, advertising SDKs,
profiling, or sale/sharing of browser data. Network data transmitted by
user-initiated unblock requests is sent only after consent and only to the
OpenPath service configured by the managed deployment.
